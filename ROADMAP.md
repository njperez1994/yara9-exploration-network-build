# ROADMAP.md
## Macana Commerce Center / Hackathon Scope Roadmap

----------------------------------------
PRIMARY HACKATHON OBJECTIVE
----------------------------------------

Deliver a focused, polished demo that shows a believable Frontier station workflow:

Docking → Wallet/Auth → Station Shell → Scan Data Flow → Data Exchange → MTC Result

This is the minimum viable hackathon story.

----------------------------------------
SUCCESS CRITERIA
----------------------------------------

The project is successful for the hackathon if it can demonstrate:

1. Immersive station entry
2. Readable and coherent station UI
3. At least one clear player utility flow
4. Credible or real connection to Sui / Stillness systems
5. A short live demo that is easy for judges to understand

----------------------------------------
PHASE 1 - ENTRY EXPERIENCE
----------------------------------------

Status target:
- Complete

Deliverables:
- docking screen
- wallet connect trigger
- authentication/loading sequence
- door opening animation
- station reveal

----------------------------------------
PHASE 2 - STATION SHELL
----------------------------------------

Status target:
- In progress

Deliverables:
- top banner with Macana branding
- corporation resource indicators:
  - LUX
  - MTC
  - scan data received
- compact tab navigation
- central module frame
- one active module at a time

----------------------------------------
PHASE 3 - CORE DEMO MODULE
----------------------------------------

Priority:
- Highest

Main deliverable:
Data Exchange module

Must show:
- scan-related data input or display
- exchange action
- result state
- visible MTC output or update

This is the module most likely to matter in judging.

----------------------------------------
PHASE 4 - WALLET / STATUS INTEGRATION
----------------------------------------

Deliverables:
- rider wallet/session state
- active address or identity indicator
- network / connection feedback
- safe fallback if live connection is unavailable

----------------------------------------
PHASE 5 - LIVE / CREDIBLE CHAIN CONNECTION
----------------------------------------

Goal:
- real or near-real Sui-connected flow

Examples:
- load data from on-chain object
- submit exchange-related action
- display live object state
- prove Stillness relevance when possible

This phase is important for Best Live Frontier Integration.

----------------------------------------
OUT OF SCOPE FOR NOW
----------------------------------------

Until the core demo loop is strong, avoid spending time on:
- full market systems
- advanced industrial crafting
- deep licensing systems
- multiple secondary modules
- cosmetic-only screens

----------------------------------------
DEMO SCRIPT TARGET
----------------------------------------

The final demo should be explainable in under 3 minutes:

1. Rider arrives at Macana
2. Access is granted
3. Station UI appears
4. Scan-related value is shown
5. Exchange action is performed
6. Result is visible in MTC / system state

----------------------------------------
NEXT DECISION RULE
----------------------------------------

If a new task does not clearly strengthen the 3-minute demo, it should probably be deferred.
